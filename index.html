Linked Issues Validator

A Jira workflow validator that checks if any linked issue matches a provided JQL query.

About

This Atlassian Connect app adds a workflow validator to Jira that blocks workflow transitions unless at least one linked issue matches the configured JQL query.

Available Endpoints

GET /create.html - Configuration page
GET /view.html - View configuration page
GET /validate.html - Validation handler

GitHub Pages Hosting

Note: This app is hosted on GitHub Pages, which only serves static files. The validation logic is simplified and relies on Jira's built-in JQL validation.

For full functionality with server-side validation, consider deploying to a hosting service that supports Node.js applications.

Usage

  1. Install the app in your Jira instance by uploading the atlassian-connect.json file
  2. Add the validator to a workflow transition
  3. Configure the JQL query that linked issues should match
  4. The transition will be allowed if any linked issue matches the JQL

Configuration Examples

  • Status-based: status = "Done"
  • Project-based: project = "SUP"
  • Complex queries: project = "SUP" AND statusCategory = "Done" AND assignee is not EMPTY
